# 04 — Animation System

## Principle

Motion should explain the portfolio’s central idea: information becomes clearer through structure. Animation is a layer of meaning, not decoration.

## Opening sequence

1. Small index/label appears.
2. Name resolves from slight blur and offset.
3. A thin mint data path draws across the composition.
4. The role and statement settle into place.
5. The user receives one clear scroll cue.

Keep it under 1.5 seconds and never block interaction.

## Scroll behaviour

- Sections reveal in a calm stagger: label, title, body, visual.
- Project media can shift slightly, never dramatically.
- Data lines may draw as the related content enters the viewport.
- Case study panels can transition between “messy input” and “clear output”.
- Tool icons should have tiny hover responses, not continuous bouncing.

## Unique interaction idea

Create a **clarity path**: a thin line travels through the site. At the hero it is an unstructured set of points; near the featured BI project it aligns into a clean chart/grid. This visual metaphor appears only in a few transitions.

## Background media

Use one short, abstract video only in the hero or featured project. It should show quiet light, data-like movement, or an abstract texture. Add a dark overlay and never let it compete with text. Provide a static poster and disable/hide the video for reduced-motion users or slow connections.

## Timing

- Micro interaction: 160–240ms.
- Section reveal: 500–800ms.
- Major transition: 900–1200ms.
- Use ease-out curves; avoid elastic/bouncy presets.

## Accessibility

All essential information must remain visible without animation. Keyboard focus must be obvious. Do not use motion as the only way to communicate state.
